# ２．マルチモーダル生成AIでのテキスト作成
→ Geminiなどの文脈を読めるマルチモーダル生成AIでテキスト化を行うアプローチ

マルチモーダル生成AIに対して「RAG」「プロンプト」「ファインチューニング」で
ハルシネーションを起こさずに原稿認識精度を上げる手法を研究する。

## patterns

### パターン1：音声＋補助テキスト（最も一般的）

 入力：
- 音声：会議の録音
- テキスト：
  ・会議の目的
  ・参加者一覧
  ・専門用語リスト
  ・アウトプット形式の指示

→ 文脈を踏まえたテキスト生成（議事録・要約など）

### パターン2：ASR結果テキスト＋元音声を同時に投入
① 音声入力
      ↓
② （省略可）ASRで文字起こし
      ↓
③ マルチモーダル生成AI
     入力：
       ・音声
       ・テキスト（ASRのラフな文字起こし）
       ・テキスト（指示・補助情報）
      ↓
④ 整形されたテキスト出力


## ASRログを使った検証フロー

① 音声取得
② ASRで生ログ生成（基準）
③ マルチモーダル生成AIで成果物作成
④ ASRログ × 生成結果の突合
⑤ 差分・リスク判定
⑥ 人 or ルールによる最終承認

ポイントは
ASRログ＝「事実の地盤」
生成結果＝「編集された成果物」
という役割分離です。

### ASRログ × 生成結果の突合（核心）

- 検証観点①：事実追加チェック（ハルシネーション）
チェック内容:
ASRログに存在しない情報が生成結果に入っていないか

🚨 NG例：
数値が増えている
断定していない内容が断定文になっている
発言していない結論が追加されている

✅ 方法
生成文の各文に対して
→ 対応するASRログ区間を紐づけ
紐づけ不可な文は 要レビュー

- 検証観点②：意味改変チェック（ニュアンス逸脱）
チェック内容

「慎重」「検討中」が「決定」に変わっていないか
仮定の話が事実扱いされていないか

例：
ASR：
「この案も一応ありかもしれない」
生成：
「この案を採用することで合意した」

🚨 高リスク差分

- 検証観点③：重要語句の保持
対象：
日付
金額
数量
固有名詞
ToDo・担当者

✅ 推奨ルール
重要語句はASRログと一致必須
一致しない場合は自動フラグ

## 「ASR × Gemini（マルチモーダル生成AI）× RAG（Retrieval-Augmented Generation）」を併用する方法

RAGは「外部の正解知識」を補い、
ASRは「発言事実」を固定し、
Geminiは「文脈理解と生成」を担う

→ 三者を分離すると 精度・検証性・拡張性が同時に成立します。

### RAGの役割を明確に分ける（重要）
RAGは 「会話の内容を増やす」ためではありません。
✅ 正しい役割

判断基準を与える
用語・定義を固定する
“どう書くべきか”を教える

❌ やってはいけない役割

発言していない事実を補う
会話の結論を勝手に導く

👉 事実は必ず ASR からのみ

### 検証フェーズでもRAGは使える
二段RAG構成（実務向け）
生成時RAG

ルール・定義を注入
表現ぶれを防ぐ

検証時RAG

「議事録チェックリスト」
「NG表現集」

### まとめ

実務向けベストプラクティス（短く）

ASR：事実の唯一のソース
RAG：定義・基準・補助ルール
Gemini：理解・再構成・生成
検証：ASR基準で実施

ここで重要なのは、LLM (Gimini)で改修したものを追加/更新した情報は許すかどうかです。

### RAGの例
- 用語・略語・正式名称の定義
SLA：
Service Level Agreement（サービス品質保証基準）。
社内では稼働率99.9%を指す（文書ID: OPS-004）。

- 議事録・要約の判断ルール

【意思決定の定義】
  ・「決めます」「確定」「〜でいきます」と明示された発言のみ
  ・ 検討・提案は意思決定に含めない

- 出力フォーマット・記載ルール

    議事録は以下の構成とする：
    1. 日時・参加者
    2. 決定事項
    3. 検討事項
    4. ToDo（担当・期限つき）

- 禁止ルール（ネガティブルール）
  禁止事項：
　・ 推測による結論の記載
　・発言されていない数値の補完
　・曖昧な決定表現の断定化


- RAGに「入れてはいけない」もの
  ・　事実・結論・決定事項
　・　解釈済みの要約
　・　こう書いてほしい答え
   
### RAG文書の理想構造（テンプレ）

```
# 概要
本RAG文書は、生成AIが議事録を作成する際の
判断基準・用語定義を提供するものであり、
事実の補完には使用しない。

# 用語定義
- ○○
- △△

# 判断ルール
- 意思決定とは〜
- 検討事項とは〜

# 出力フォーマット
- 見出し構成
- 書式ルール

# 禁止事項
- 未発言事実の追加
- 推測表現
```
