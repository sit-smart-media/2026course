# 二段検証

**WhisperX を「位置・存在証明器」**として使い、
**CTC‑ASR を「音響的可能性の裁判官」**として使う構成

つまり：

WhisperX：
👉 その単語は音声のどこに対応しているか？
CTC‑ASR：
👉 その単語列は音声から生成可能か？

を 役割分離で検証します。

# なぜ WhisperX 単独では不十分なのか
WhisperX は非常に強力ですが：
✅ 強い

word-level alignment
時間的一貫性
「存在しない単語」を検出できる

❌ 弱い

音響尤度を返さない
「この語が“生成可能か”」を否定できない
attention decoder なので 無音でも出力可能

👉
「追加語は検出できるが、無理な置換は止めきれない」

# なぜ CTC-ASR を組み合わせるのか
CTC 系 ASR（wav2vec2, Conformer‑CTC など）は構造的に：

✅ 音声フレームなし → 文字生成不可
✅ 強い monotonic alignment 制約
✅ blank に逃げるが 捏造はしない

つまり：

CTC = 物理的制約付きモデル

これが LLM・Whisper 系モデルの
「自由生成バグ」を止める 最後の安全装置になります。

# 二段検証 パイプライン(推薦構成)
Audio
 ↓
Stage 1: WhisperX
  - transcription
  - word timestamps
  - alignment confidence
 ↓
Error Detection
  - 追加語候補
  - 不自然な長区間
 ↓
LLM（制約付き修正）
  - 追加禁止
  - 置換・削除のみ
 ↓
Stage 2: CTC-ASR 검증
  - 同テキスト forced alignment
  - CTC score / alignment feasibility
 ↓
Accept / Reject / Rollback

## Stage 1：WhisperX（存在検証）
WhisperX では以下をチェックします：
✅ 検証①：単語の「存在」

追加された単語に time span が存在するか
無音区間に語が割り当てられていないか

✅ 検証②：時間的一貫性

既存単語が大きく位置変更されていないか
話者区間をまたいでいないか

👉
WhisperX =「証拠映像」

LLM の役割（超限定）
LLM は 絶対に自由生成させない。
許可する操作

誤認識語の置換
冗語・誤挿入の削除

禁止

単語追加
意味補完
要約・言い換え

プロンプトの鉄則
- You may only replace or delete existing words.
- You MUST NOT introduce new lexical items.
- Explain which original word you changed and why.

## Stage 2：CTC-ASR（音響可能性検証）
ここが WhisperX ではできない部分。
方法 A：forced alignment

CTC モデルに LLM後テキストを固定
alignment が破綻 → reject

方法 B：CTC score 比較

修正前 vs 修正後
尤度が大幅悪化 → hallucination の疑い

方法 C：phoneme レベル

grapheme → phoneme
対応フレーム確認

👉
CTC =「その発音、物理的に可能？」

# この二段検証で防げる失敗パターン
❌ WhisperXだけでは止まらない例
Audio:    "... chest pain ..."
ASR:      "chest pain"
LLM:      "severe chest pain"   ← 置換なので WhisperX を通る


"severe" が既存語に“吸着”して見える

✅ CTC-ASR が止める

/sɪˈvɪr/ の音素が存在しない
alignment 不可能 → reject


# 実運用・研究でのメリット
✅ 強い保証ができる

「音声に存在しない語は出ません」
医療・議事録・講義で重要

✅ LLMを安全に使える

提案器としてのみ使う
決定権は ASR 側

✅ 論文的に説明しやすい

役割分離
物理制約の導入
hallucination 対策として筋が通る

# 注意点（重要）
⚠ WhisperX と CTC で tokenizer は一致しない

subword / grapheme のズレを吸収する設計が必要

⚠ CTC を「主 ASR」にしない

認識性能は Whisper 系が上
CTC は 検証専用

⚠ 処理遅延

real‑time ではなく batch / semi‑offline 向け
