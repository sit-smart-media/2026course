
# introduction

医用音声（診療録・退院サマリ・カルテ入力）向けに特化した
hallucination‑safe loop を、
設計原理 → 具体構成 → 制御ルール → なぜ安全か → 実装時チェックリスト
の順でまとめたものです。
※前提：WhisperX + CTC‑ASR + LLM 二段検証。

# 医用音声で「hallucination が絶対に許されない」理由
医用音声では ASR/LLM の誤りがそのまま：

診断の捏造
症状の過剰／欠落
薬剤・用量の誤記
否定文の反転（denies → has）

につながります。
特に危険なのは：

LLMが「医療的にもっともらしい情報」を補完してしまうこと

これは一般的な hallucination より 質的に危険 です。

# 医用 hallucination‑safe loop の設計原理（最重要）
原理 1：
観測（音声）に存在しない情報は 1 字符も出さない
原理 2：
LLMは“提案”しかできず、“確定”はできない
原理 3：
否定・数値・薬品名は最優先で検証
→ この3原理を 構造で強制 します。
（プロンプトだけでは不十分）

# 各ステージの「医用向け」ポイント

(1) WhisperX：基礎証拠生成
WhisperX の役割は 一次証拠。
医用向けに必須なこと

話者分離（Doctor / Patient）
単語単位 timestamps
無音区間の明示

✅ 医用 hallucination の多くは
「無音・相槌・間」から始まる

(2) 医用 Error Detection（LLM前）
LLMに渡す前に 必ずフラグ化 する語群：
🔴 最優先

否定語：no / denies / without / negative
数値：血圧、量、日数、mg, ml
薬剤名：一般名＋商品名
診断語：myocardial infarction 等

👉 これらは 「触るときに理由必須」。

(3) 医用 LLM（STRICT MODE）:絶対ルール（医用）
You MUST NOT:
- add symptoms
- infer diagnosis
- normalize numbers
- expand abbreviations

You MAY:
- correct homophones
- fix partial words
- delete obvious misrecognitions

For every change:
- point to original tokens
- explain acoustical reason

!重要

要約・言い換え = 即違反
医学常識の利用 = 即違反

👉 LLMの知識は 使わせない。

(4) WhisperX 再アライメント（存在検証）
医用で特に見る点

修正語が 同一 time span に収まっているか
否定語の位置が変わっていないか
数値が別区間に「滑り込んで」いないか

❌ 例（即 reject）：
original: "denies chest pain"
LLM:      "denies severe chest pain"

理由：

"severe" に対応する acoustic span が無い

(5) CTC-ASR（二次裁判官）
医用での役割

Whisper 系では止められない
無理な置換 を止める

特に効くケース

数値・薬品名の音素不整合
「no / known / now」などの致命的混同
長語の医療用語

CTC forced alignment が破綻 → ❌

# この loop で防げる医療事故タイプ
|事故タイプ|WhisperX|CTC|結果|
|---|---|---|---|
|症状の追加|✅|✅|防止|
|薬剤名捏造|✅|✅|防止|
|否定反転|✅|✅|防止|
|数値補完|⚠|✅|防止|
|医学知識補完|✅|✅|防止|


# 評価指標（医用向け）
通常の WER は使いません。
医用必須指標

Negation Error Rate
Numerical Consistency Rate
Medication Hallucination Rate
Hallucination Error Rate (HER)

👉
「正しく言っていない」より
「言っていないことを言う」方が重罪

# 実装チェックリスト（現場向け）
✅ LLMが 単語総数を増やしていないか
✅ 否定語の index が変わっていないか
✅ 数値が正規化されていないか
✅ CTC alignment が 100% 通っているか
✅ 通らない場合は 必ず human review

# まとめ（医用向けの一言）

医用音声での LLM は
「書き手」ではなく
「誤字脱字校正専用ツール」
であるべき。

WhisperX が 証拠提示、
CTC-ASR が 物理制約、
LLM は 補助者。
この順序を崩した瞬間に 医療事故リスクが出ます。
