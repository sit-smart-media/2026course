# instroduction
専門用語辞書は必須だが「使い方」を間違えると hallucination を増やします。
したがって医用音声では、

✅ 辞書は「生成を助けるため」ではなく
✅ 「生成を縛るため（検証・制約）」に使う

のが正解です。
以下で 安全な併用設計 を体系的に説明します。

# 1. なぜ医用音声に「専門用語辞書」が必要か
医用音声で起きる典型的なASR誤りは：

同音異義（ileum / ilium）
音素的に長い語（myocardial infarction）
一般語への吸収（metformin → “medicine”）
略語（BP, HR, SOB）

つまり：

辞書なし＝誤認識
辞書を自由生成に使う＝幻覚増幅

という二面性があります。

# 危険な辞書併用（やってはいけない）
まず NG設計 を明確に。
## NG①：LLMプロンプトに「この辞書を使ってね」
Plain TextUse the following medical dictionary to fix the transcript.

Use the following medical dictionary to fix the transcript.

なぜ危険か

LLMは「辞書を参照＝使うべき」と解釈
音声に無い医療語を補完
「医学的にそれらしい hallucination」が発生

👉 医療における最悪の失敗例。

## NG②：Whisper / ASR に辞書バイアスを強くかける

hotword boost
医療語を優先補正

結果：

無音区間に専門語が出る
CTC であれば防げるが Whisper 系では危険

# 正しい思想：辞書は「裁判官側」に置く
原則

辞書は生成器（ASR / LLM）ではなく
検証器（Verifier）にだけ使う

→ 医療 hallucination-safe 設計の核心。

# 辞書の安全な使い方（3分類）
## ① 存在検証（Existence check）✅最重要
目的

LLMが出した専門用語は「音声に存在したか？」

方法

辞書語が含まれる場合：

WhisperX：time span があるか
CTC：phoneme alignment が可能か


✅ 通る → 使ってよい
❌ 通らない → hallucinationとして reject

## ② 置換制約（Constrained substitution）
辞書を「置換候補集合」として使う。
例
ASR: "metformin"
候補: [metformin, metoprolol, morphine]

重要

追加は禁止
あくまで既存トークンの 音素的近傍 のみ

→ CTC alignment で裁く。

## ③ 危険語フラグ（High‑risk vocabulary）
辞書は 触ってはいけない語群 を定義するためにも使う。
医用で必須

薬剤名
用量（mg/ml）
否定語とそのスコープ
診断名

👉 LLM がこれらを変更した場合：

理由説明必須
CTC + alignment 両方通過必須
一つでも失敗 → human review

# 実装例（ルール化）
If a token is in MEDICAL_DICTIONARY:
  - LLM cannot add this token
  - Replacement requires:
      * same time-span alignment
      * successful CTC forced alignment
Else:
  - Dictionary NOT consulted
  
重要

辞書は 「照会」されるだけ
LLMには 辞書を見せない

# よくある誤解（重要）
❌「辞書を入れれば賢くなる」
→ ❌ 幻覚が医学的になるだけ
✅「辞書で縛れば安全になる」
→ ✅ 検証器としてのみ正解

# まとめ（医用向けの鉄則）

医用音声において
専門用語辞書は
「知識」ではなく
「制約」として使え。


WhisperX：存在証拠
CTC-ASR：物理制約
辞書：語彙制約
LLM：局所修正のみ

この役割分離を守れば、
hallucination-safe な医用音声記録が成立します。
