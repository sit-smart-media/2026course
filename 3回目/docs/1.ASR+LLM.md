**ASR＋LLM 協調設計（co-design / co-processing）** 「信頼性（reliability）」が最大の課題

①概念整理 → ②信頼性問題の技術的本質 → ③代表的論文 → ④Hugging Faceモデル → ⑤GitHub実装

という研究・実装の両面から体系的にまとめます。

1. ASR + LLM 協調設計とは何か
  ASR + LLM 協調設計とは、音声認識（ASR）と大規模言語モデル（LLM）を
単なる直列パイプラインではなく、役割分担・相互補完を前提に共同最適化する設計思想。

従来：単純パイプライン
音声 → ASR → テキスト → LLM → 出力

問題：
ASR誤りがそのままLLMへ伝播
LLMが「それらしく」補完し、誤情報を確信的に生成

協調設計：代表的パターン

| パターン |　説明 |
| --- | --- |
| N-best + LLM | ASR仮説集合を使い、LLMで再選択・修正 |
| Confidence-aware | ASR信頼度に応じてLLMの介入度を制御 |
| Joint / Synchronous | ASRとLLMを内部表現レベルで統合 |
| Verification loop | LLM出力をASRや音響的根拠で再検証 |

2. なぜ「信頼性」が最大の課題になるのか

2.1 二重の不確実性が累積する
ASR + LLMでは、2種類の不確実性が連鎖します。

ASRの不確実性

雑音、話者差、方言、分布外音声
→ 欠落・置換・幻覚（無音からの生成）

LLMの幻覚（Hallucination）

入力テキストを「事実」と仮定
文法的・意味的に“もっともらい”補完を行う

この結果、
ASRの小さな誤りが、LLMによって意味的に増幅される

ことが最大のリスクになります。
この問題は、音声基盤モデルにおけるASR Hallucinationとして定量的にも示されています。 [aclanthology.org]

2.2 WERでは測れない「危険な誤り」
近年の研究では以下が指摘されています：

低WERでも致命的幻覚が存在
WER/CERは「生成してはいけない情報」を評価できない

そこで Hallucination Error Rate (HER) のような新指標が提案されています。 [aclanthology.org]

2.3 LLMによる ASR 修正は「正解を壊す」ことがある
LLMベースASR誤り訂正は強力ですが、

正しい単語まで書き換える
文脈的に自然な「誤情報」を生成

というリスクがあります。
これを直接扱った研究では、

「LLMをそのまま使うと、正しい部分を壊す hallucination が発生する」

ことが明示されています。 [arxiv.org]
