    1. reference 作成: 01_asr_raw.txt を手修正 → results/04_reference.txt に保存（正解作り自体が学び）
    2. variant ループ:
      - --variant strict / with_glossary / fewshot / reflect を順に実行
      - --all で一括も可
    3. CSV 分析: 04_wer_log.csv の CER/WER/len_ratio を比較し、どの prompt が効いたか定量判断
    4. prompt 改造: 自分で新 variant 追加（用語追加・few-shot 例追加・制約変更）→ 再測定
    5. レポート: 改善理由を仮説→実験→結果の流れで記述

    共通の学び所

    - LLM 出力を盲信せず検証ループで確認する設計思考
    - 主観評価でなく CER/score で意思決定する習慣
    - prompt は仮説実験の単位であり、変えたら必ず測る
