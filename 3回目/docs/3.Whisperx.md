# WhisperX の本質（検証観点）
WhisperX は単なる ASR ではなく、

Whisper（attention-based ASR）
強力な forced alignment
単語・タイムスタンプ・話者分離

を提供する 「後付け検証に強い ASR」 です。
重要なのは：

WhisperX は
「この単語は、いつ・どこに対応しているか」
を非常に正確に示せるが、
「この単語が音声的に生成可能か」
をスコアで厳密に返す設計ではない

という点です。

# Error Detection → Correction → Verification loop
WhisperX の強み：

- 単語単位タイムスタンプ
- alignment confidence
- 話者整合性

これにより：

LLMが修正した単語が
- 音声中のどの区間にも対応していなければ即 reject
- 音声区間を削除・移動した不正編集を検出可能

具体的にできること
LLM 修正前:"the patient denies chest pain"
LLM 修正後:"the patient denies severe chest pain"その他の行を表示する

WhisperX 検証：
"severe" に対応する time span が存在しない
→ ❌ hallucination 検出

✅ WhisperXは「追加語」に極めて敏感
👉 これは LLMの幻覚を止めるのに最強の検証軸

# Confidence-aware control loop

WhisperX は implicit な token confidence を持つ
明示的 probability ほど強くないが、

long silence,
uncertain alignment,
話者切り替え不整合
などを confidence proxy に使える

👉
「この区間だけ LLM 介入を許可する」
という ゲーティング制御に向く。

# 推薦構成

Audio
 ↓
WhisperX
  - text
  - word timestamps
 ↓
Error Detection
  - no-alignment words
  - long silence hallucination
 ↓
LLM (局所修正のみ)
  - 追加禁止
  - 置換は time-span 内のみ
 ↓
WhisperX Re-alignment
 ↓
Reject / Accept

## LLM に課すべきルール（超重要）

❌ 新語追加禁止
✅ 既存語の置換・削除のみ
✅ 変更語の理由を出力させる
✅ WhisperX alignment が通らなければ棄却

👉 これで LLM hallucination はほぼ封殺できます。

## 研究的に重要な一言

WhisperX は
「音声に存在しない単語を“証拠付きで否定”できる」
が、
「音声的確率で裁く裁判官」ではない

だから：

WhisperX × LLM = 検証型 loop
CTC-ASR × LLM = 音響制約型 loop

という使い分けが正解です。
