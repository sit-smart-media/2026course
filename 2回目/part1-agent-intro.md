# Part 1：エージェントとは何か？（0〜15分）

---

## 1-1. 通常のプログラムとエージェントの違い

通常のプログラムは「入力 → 処理 → 出力」という**固定されたフロー**で動きます。

```
入力: "2 + 3 は？"
処理: 決まった計算式
出力: "5"
```

**エージェント**は違います。目標（ゴール）だけ与えられ、そこに到達するための**行動を自分で決めます**。

```
ゴール: "このバグを直して"
エージェント:
  1. コードを読む（ツール使用）
  2. エラーを調べる（ツール使用）
  3. 修正案を考える（LLM による推論）
  4. テストを実行する（ツール使用）
  5. 完了を報告する
```

---

## 1-2. コーディングエージェントの構成要素

```
┌─────────────────────────────────┐
│           LLM（頭脳）            │
│  「次に何をすべきか」を判断する    │
└────────────┬────────────────────┘
             │ 指示
     ┌───────▼────────┐
     │   ツール群       │
     │ ・ファイル読み書き │
     │ ・コード実行      │
     │ ・Web検索        │
     │ ・コマンド実行    │
     └────────────────┘
```

LLM が「どのツールを・どの順番で・どんな引数で呼ぶか」を決めます。  
このループを繰り返すことで複雑な問題を解きます。

---

## 1-3. ReAct パターン（最も基本的な動き方）

エージェントの思考サイクルを **ReAct** と呼びます。

```
Thought（考える）→ Action（行動）→ Observation（観察）→ Thought → ...
```

**例：「tests/test_calc.py を実行して失敗しているテストを直して」**

```
Thought: まずテストを実行して現状を確認しよう
Action: run_command("pytest tests/test_calc.py")
Observation: FAILED test_divide - ZeroDivisionError

Thought: divide 関数でゼロ除算チェックが抜けている
Action: read_file("calc.py")
Observation: def divide(a, b): return a / b

Thought: b == 0 のチェックを追加すればよい
Action: edit_file("calc.py", ...)
Observation: ファイルを更新しました

Thought: テストを再実行して確認しよう
Action: run_command("pytest tests/test_calc.py")
Observation: PASSED ✓

Thought: 修正完了
```
