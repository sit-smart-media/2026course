## 1. イントロダクション：なぜバージョン管理が必要か（10分）

### バージョン管理がない世界

```
レポート_最終版.docx
レポート_最終版2.docx
レポート_最終版_本当の最終版.docx
レポート_最終版_提出用_修正済み.docx
```

- ファイル名で管理 → どれが最新か分からなくなる
- チームで作業 → 誰がどこを変えたか追えない
- 間違えた → 元に戻せない

### Gitとは

- **Git**：ファイルの変更履歴を記録・管理するツール（ローカルで動作）
- **GitHub**：Gitの履歴をインターネット上で共有・管理するサービス

### Gitで解決できること

| 問題 | Gitによる解決 |
|------|--------------|
| どれが最新か分からない | 変更履歴が時系列で自動記録される |
| チームで同時編集すると壊れる | ブランチで並行作業、マージで統合 |
| 間違えたら戻せない | いつでも過去の状態に戻せる |
| 誰が何を変えたか不明 | コミット履歴で全て追跡可能 |

## 2. GitHubアカウントの作成（15分）

### 手順

1. ブラウザで [https://github.com](https://github.com) にアクセス
2. 右上の **「Sign up」** をクリック
3. 以下の情報を入力：
   - **Email**：大学のメールアドレス（`xxxxx@sit.shonan-it.ac.jp`）を推奨
     - 後でEducation特典の申請に必要
   - **Password**：8文字以上、英数字と記号を組み合わせる
   - **Username**：今後ずっと使う名前。本名またはハンドルネーム
     - 例：`taro-yamada`、`sit-taro2026`
4. メール認証：届いたメールのリンクをクリック
5. 初期設定の質問に回答（スキップ可）

### 確認ポイント

- [ ] GitHubにログインできる
- [ ] プロフィールページ（`github.com/自分のユーザー名`）が表示される

### プロフィールの設定（推奨）

1. 右上のアイコン → **Settings**
2. **Profile** で以下を設定：
   - **Name**：表示名（日本語OK）
   - **Bio**：一言紹介（例：「湘南工科 情報学科 3年」）
   - **Profile picture**：顔写真やアイコンを設定

## 3. GitHub Education（学生特典）の申請（15分）

### GitHub Education とは

学生であれば無料で利用できる特典パッケージ：

- **GitHub Pro**（通常$4/月）が無料
  - プライベートリポジトリの高度な機能
- **GitHub Copilot**（AIコード補完）が無料
- その他、多数の開発ツールが無料で利用可能

### 申請手順

1. [https://education.github.com/students](https://education.github.com/students) にアクセス
2. **「Join GitHub Education」** をクリック
3. GitHubにログイン済みであることを確認
4. 以下の情報を入力・提出：
   - **学校名**：`Shonan Institute of Technology` を選択
   - **メールアドレス**：大学メール（`@sit.shonan-it.ac.jp`）
   - **学校での利用目的**：簡単に英語で記載
     - 例：`For coursework and team project collaboration`
   - **証明書類**：以下のいずれかをアップロード
     - 学生証の写真
     - 在学証明書
     - 大学メールのスクリーンショット
5. **「Submit」** をクリック

### 注意事項

- 審査に数日〜2週間かかる場合がある
- 承認されるとメールで通知が届く
- 今日申請しておけば、次回以降Copilotなどが使えるようになる

### 確認ポイント

- [ ] Education申請を送信できた
- [ ] 大学メールアドレスがGitHubに登録されている

## 4. GitHub Desktopのインストールと初期設定（15分）

### GitHub Desktop とは

- GitをGUI（グラフィカル）で操作できるアプリ
- コマンドラインを使わずに、クリック操作でGitの基本操作ができる
- 初心者に最適

### インストール手順

1. [https://desktop.github.com](https://desktop.github.com) にアクセス
2. **「Download for Windows」**（または macOS）をクリック
3. ダウンロードしたインストーラーを実行
4. インストール完了後、アプリを起動

### 初期設定

1. **「Sign in to GitHub.com」** をクリック
2. ブラウザが開くのでGitHubにログイン → **「Authorize」** をクリック
3. **Configure Git** 画面：
   - **Name**：コミットに表示される名前（本名推奨）
   - **Email**：GitHubに登録したメールアドレス
4. **「Finish」** をクリック

### 確認ポイント

- [ ] GitHub Desktopが起動する
- [ ] GitHubアカウントと連携できている（左上にユーザー名が表示）

## 5. GitHub基本操作の実習（25分）

### 実習の流れ

```
リポジトリ作成 → ファイル追加 → コミット → プッシュ → 確認
```

### 5-1. リポジトリの作成（5分）

**リポジトリ（Repository）** = プロジェクトのフォルダ。ファイルと変更履歴をまとめて管理する場所。

1. GitHub Desktopで **File → New Repository** を選択
2. 以下を入力：
   - **Name**：`my-first-repo`
   - **Description**：`初めてのGitHubリポジトリ`
   - **Local Path**：保存先フォルダ（デスクトップなど分かりやすい場所）
   - **Initialize with README**：チェックを入れる
3. **「Create Repository」** をクリック

### 5-2. ファイルの作成と編集（5分）

1. GitHub Desktopで **「Show in Explorer」**（Macは「Show in Finder」）をクリック
2. フォルダが開くので、テキストエディタで `README.md` を開く
3. 以下の内容に編集して保存：

```markdown
# 自己紹介

## 名前
（自分の名前を書く）

## 学年・学科
（学年と学科を書く）

## 興味のある技術
- （例：AI、Web開発、ゲーム開発など）

## この授業で学びたいこと
- （自由に書く）
```

### 5-3. コミット（5分）

**コミット（Commit）** = 変更を記録すること。「セーブポイント」のようなもの。

1. GitHub Desktopに戻ると、変更が検出されている（左側に差分が表示）
2. 左下の入力欄に**コミットメッセージ**を書く：
   - **Summary**：`自己紹介を追加`
3. **「Commit to main」** をクリック

### 5-4. プッシュ（5分）

**プッシュ（Push）** = ローカルの変更をGitHub（インターネット上）にアップロードすること。

1. 上部の **「Publish repository」** をクリック
2. 設定画面：
   - **Name**：そのまま（`my-first-repo`）
   - **Keep this code private**：チェックを外す（公開にする）
3. **「Publish Repository」** をクリック

### 5-5. GitHubで確認（5分）

1. GitHub Desktopで **Repository → View on GitHub** をクリック
2. ブラウザでリポジトリページが表示される
3. 確認すること：
   - `README.md` の内容が表示されているか
   - コミット履歴が見えるか（「1 commit」をクリック）

### 確認ポイント

- [ ] リポジトリが作成できた
- [ ] ファイルを編集してコミットできた
- [ ] GitHubにプッシュして、ブラウザで確認できた

## 6. まとめと次回予告（10分）

### 今日学んだこと

| 用語 | 意味 |
|------|------|
| **Git** | ファイルの変更履歴を管理するツール |
| **GitHub** | Gitの履歴をオンラインで共有するサービス |
| **リポジトリ** | プロジェクトのフォルダ（履歴付き） |
| **コミット** | 変更を記録する（セーブポイント） |
| **プッシュ** | ローカルの変更をGitHubにアップロード |

### 今日の成果

- GitHubアカウントを作成した
- GitHub Educationに申請した
- GitHub Desktopをインストールした
- 初めてのリポジトリを作成し、コミット・プッシュを行った

### 課題（次回までに）

1. **GitHub Educationの承認を確認する**
   - 承認メールが届いたら、GitHub Copilotを有効にしてみる
2. **リポジトリに追加のコミットを行う**
   - `README.md` に「好きなプログラミング言語」の項目を追加してコミット・プッシュする
3. **他の受講者のリポジトリを見てみる**
   - クラスメイトのGitHubプロフィールを訪問してみる

### さらに

- **ブランチとマージ**：チームで同時に作業するための仕組み
- **プルリクエスト**：変更をレビューしてもらう方法
- **プロジェクト課題の選定**：本コースで取り組む課題を決める
